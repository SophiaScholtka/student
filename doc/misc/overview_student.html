jCrypt ist im Rahmen einer Studienarbeit im Wintersemester 2009/2010 und den ersten Wochen des Sommersemesters 2010 entstanden.

Hinweise zur Einrichtung der Programmierumgebung

  1. Entpacken Sie das Archiv jCrypt_1.1_student.tar.gz bzw. das Archiv jCrypt_1.1_student.zip in ein beliebiges Verzeichnis (im folgenden mit student bezeichnet).
  2. Starten Sie Eclipse.
  3. Wählen Sie im Workspace Launcher (File → Switch Workspace{... | → Other...}) als Workspace das Verzeichnis student aus und bestätigen Sie mit <OK>.
  4. Wählen Sie im Import-Dialogfenster (File → Import...) als Quelle die Kategorie General/Existing Projects into Workspace aus und bestätigen Sie mit <Next>. Geben Sie als Root Directory das Verzeichnis student an und schließen Sie den Import mit <Finish> ab. Die Quelltextdateien sollten im Anschluß an den Import automatisch fehlerfrei kompiliert werden (jCrypt ist entworfen für Java 1.5).
  5. Alle Dateien von jCrypt sind kodiert mit UTF-8 (jCrypt_1.1_student.tar.gz) oder CP1252 (jCrypt_1.1_student.zip). Damit diese Dateien in Eclipse richtig dargestellt werden, muß im Preferences-Dialogfenster (Window → Preferences[...]) in der Kategorie General/Workspace das Text File Encoding entsprechend eingestellt sein. Ggf. ist dort auch der Text File Line Delimiter zu ändern. Möchten Sie die jCrypt-Dateien in eine andere Kodierung konvertieren, probieren Sie vielleicht libiconv.
  6. In jCrypt.properties sind vier Encodings einzustellen. Unter Linux könnte UTF-8, unter Microsoft Windows CP1252 eine gute Wahl sein. Bitte beachten Sie, daß Eclipse auch eine eigene Konsolenkodierung benutzt. Diese kann zu jeder Run Configuration (Run → Run[ Configurations]...) in der Kategorie Java Application auf der Karteikarte mit dem Reiter Common im Bereich Console Encoding geändert werden.
  7. Testen Sie die Programmierumgebung mit der Run Configuration 90 CharacterMapping programmierer.alph. Sie sollten etwa folgende (hier deutlich gekürzte) Ausgabe erhalten:

Die Verzeichnisstruktur von jCrypt

Die Verzeichnisstruktur ist unterteilt in Verzeichnisse, die den einzelnen Praktikumsaufgaben zugeordnet sind (diese beginnen mit den Ziffern 00 bis 05) und die folgenden jCrypt-Systemverzeichnisse:

alphabet
Standardverzeichnis für benutzerdefinierte Alphabete.
batch
Batch-Dateien für Microsoft Windows, die demonstrieren sollen, wie man jCrypt auch von der Eingabeaufforderung nutzen kann.
doc
Dokumentation von jCrypt (Javadoc).
jCrypt
Core-Files, Launch Configurations und Umgebungseigenschaften von jCrypt.
shell
Shell-Skripte für Linux, die demonstrieren sollen, wie man jCrypt auch von der Command Line nutzen kann.
table
Standardverzeichnis für Häufigkeitstabellen.
text
Standardverzeichnis für ausgewählte Textdateien:
