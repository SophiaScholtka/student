Programmierumgebung für kryptographische Protokolle

docs/ Dokumentationen der für den Aufgaben-Teil II zur Verfügung gestellten Klassen.
classes.jar Archiv der für den Aufgaben-Teil II zur Verfügung gestellten Klassen.
Damit die darin enthaltenen Klassen sowohl beim Kompilieren als auch beim Starten eigener Klassen gefunden werden, muß es in der Umgebungsvariablen CLASSPATH enthalten sein.
local/ Diese Dateien sollen von jeder Gruppe in Ihr Arbeitsverzeichnis kopiert werden; beispielsweise durch den Aufrug des Shellscriptes
$KRYPTOUMGEBUNG/protokoll/make_local.sh
aufruft. Das Verzeichnis enthält die folgenden Dateien:
  1. server.sh
    Bevor ein Protokoll ausgeführt werden kann, muß ein Server gestartet werden, der die Kommunikation zwischen den teilnehmenden Parteien (Clients) gewährleistet. Da dieser Server mehrere Protokolle gleichzeitig verwalten kann, braucht nur ein Server pro Host gestartet zu werden. server.sh ist ein einfaches Shellscript, welches den Server mit dem erforderlichen vollständigen Klassennamen startet.

  2. client.sh
    Eine .class-Datei, die den Ablauf eines Protokolls beschreibt, wird ausgeführt, indem jede teilnehmende Partei einen Client startet und in einem der sich dann öffnenden Eingabefenstern den Namen dieser .class-Datei auswählt. client.sh ist ein einfaches Shellscript, welches den Client mit dem erforderlichen vollständigen Klassennamen startet.

  3. Skeleton.java
    Hierin sind die leeren Methoden sendFirst und receiveFirst mit den für das jeweilige Protokoll spezifischen Algorithmen auszufüllen. Die Klasse Communicator stellt hierfür Methoden zur Kommunikation mit den anderen Parteien zur Verfügung.

  4. example.java
    Ein Beispielprotokoll, das als nicht wörtlich zu nehmendes Beispiel dienen mag, wie eigene Protokolle anzugehen sind.
